# PIZZA-HUT-SALES-ANALYSIS-SQL


 Project Overview
 
This project focuses on analyzing Pizza Hut sales data using SQL to uncover meaningful business insights. The analysis helps understand customer behavior, sales trends, and product performance, enabling data-driven decision-making.

Objectives

Analyze overall sales performance
Identify top-selling and most profitable pizzas
Understand revenue trends by category and date
Analyze customer and city-wise sales performance
Support business decisions using data insights

Business Background

Pizza Hut operates in a highly competitive food industry where understanding customer preferences and sales trends is crucial.

This project helps management:

Optimize menu offerings

Improve sales strategies

Enhance customer satisfaction

Increase revenue through data insights

Dataset Description

The dataset consists of multiple relational tables:

Tables Used

Orders → Order-level information

Order_Details → Quantity and pizza mapping

Pizzas → Pizza details (name, category, price)

Customers → Customer details (city, contact info)

 Key Columns

order_id, order_date, customer_id
pizza_id, pizza_name, category, price
quantity, total_amount, city

Tools & Technologies

SQL (MySQL / PostgreSQL)

Excel / CSV (Data Source)

(Optional) Power BI / Tableau for visualization

 SQL Concepts Used
 
Joins (like INNER JOIN),
Aggregations (SUM, COUNT),
GROUP BY,
ORDER BY,
Filtering & Sorting,
Subqueries (optional extension)
